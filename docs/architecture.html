ARGUS architecture and flows

Interactive map of the diligence engine: click any system to light its connections, search anything, or trace a named flow. Machine twin for agents: docs/architecture.json.

Invariants a change must never break

  • H2: score floors come only from strictly verified facts; corroboration, press, and attestation widen ceilings, never mint minimums.
  • Never-waive screens (identity, OFAC name and address, trust-graph, founder-asset) must be recorded for clearance; the trust-graph screen owns a reserved 60s budget window.
  • resolved_name is never inferred; legal and sanctions screens require a provider-resolved real person.
  • Sanctions and legal run live every audit - never cached in the knowledge base.
  • Model leads never route roles, trigger caps, or floor scores.
  • Report versions are immutable (DB triggers); every audit runs in an isolated cost ledger and cannot persist without one.
  • Budget arithmetic: collection reserve minus graph-screen reserve must cover the 180s analyst window (250-60=190).
  • Bundle rule: API routes import api/_collector.js; after editing server/, run npm run build.
  • Deploy only by pushing main; no cron - the watchlist sweep is manual by design.
  • No em or en dashes in authored copy (CI-enforced three ways).
