Rules a change must never break

  • No score without proof. Minimum scores come only from strictly verified facts; corroboration, press, and self-claims can raise the ceiling but never guarantee points.
  • Safety screens are mandatory. Identity, sanctions (name and wallet), bad-actor graph, and founder-asset checks must all run before a report is called complete.
  • Names are never guessed. Legal and sanctions screening only runs on a provider-verified real person.
  • Sanctions run fresh every time. Never served from memory, because someone can be sanctioned tomorrow.
  • AI suggestions are leads, not evidence. They never pick the methodology, cap a score, or set a floor.
  • Reports are immutable. Once saved they cannot change; every run carries its own cost ledger and cannot save without one.
  • Time is budgeted. Evidence gathering stops early enough that scoring and the bad-actor screen always get their full window.
  • One build path. Server code ships inside one bundle; deploys happen only by pushing main.
